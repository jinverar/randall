# Randfuzz Ghidra scripts

First-class coverage stalking inside Ghidra — **this is the primary integration**.
[Dragon Dance](https://github.com/0ffffffffh/dragondance) is optional (binary drcov only).

## Install

1. Open Ghidra → **Window → Script Manager**
2. **Script Directories** (toolbar) → add this folder: `tools/ghidra` from the Randfuzz repo  
   (or copy `*.py` into your user scripts directory)
3. Refresh the script list — scripts appear under category **Randfuzz**

## Scripts

| Script | Purpose |
|--------|---------|
| `RandfuzzImportEdges.py` | Prompt for `coverage_edges.txt`; paints BBs; honors sibling `modules.txt` |
| `RandfuzzImportLayers.py` | Pick and run a generated `*_stalk_layers.py` / `ghidra_import.py` |

Generated by Randfuzz (do not edit by hand for long):

| Source | Output |
|--------|--------|
| `randall stalk export -p P --format ghidra` | `data/stalk/P/export/*_stalk_layers.py` |
| `randall stalk ghidra-pack -p P` | layers + importers + `DRAGON_DANCE.txt` + binary sidecars |
| `randall stalk dynapstalker LOG EXE out.py --format ghidra` | one-shot layer script |
| `randall export -i <crash-guid>` | `data/exports/<id>/ghidra_import.py` (novel vs baseline + goTo) |

## Dragon Dance (optional)

Dragon Dance imports **binary** `drcov` logs (`drrun -t drcov` **without** `-dump_text`).
Randfuzz fuzzing uses `-dump_text` so our parser and these scripts work.

```bash
# During fuzz (file targets):
#   fuzz.captureBinaryDrcov: true  →  data/corpus/<project>/traces-binary/

# One-shot:
randall stalk capture-binary -p <project> [-i seed.bin]
```

Then import `traces-binary/*.log` (or packed `binary_*.log`) in the Dragon Dance window.
See [docs/GHIDRA_INTEGRATION.md](../../docs/GHIDRA_INTEGRATION.md).
