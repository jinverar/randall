        Scare Floor — case recipes
        
          
        Prep scare attempts (test cases), save reusable recipes, queue them as seeds, then run the shift on Campaign.
        
          
          
            
        1. Create or pick a target — the project name becomes the entry in
              Fuzz → Campaign → Target profile.
          2. 
            
        2. Build a recipe — stack blocks, use a preset, or upload a valid sample. Save the recipe to reuse or combine later.
          3. 
            
        3. Queue seeds — Preview → Save as seed (or Save exact sample). Multiple seeds = multiple cases in one campaign.
          4. 
            
        4. Tune mutators → Campaign → Start — the floor runs every queued seed through the mutators.
          5. 
          
        
          
        Network targets talk to host:port. File-format targets feed a temp file to an executable (no host/port).
